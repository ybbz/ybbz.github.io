我在新服务器配置上走过的那些坑(1)

发表于 2016-03-18   |   分类于 Linux   |     |   阅读次数

昨天在自己的MBA上跑算法的时候，机器竟然发烫了；这可真不得了，在我的印象中，除了为数不多的几次视频剪辑，我的本儿就没发过热。导师问我，要不要去弄弄5楼闲置的新服务器，一直在地上放着，还没装系统；好吧，这个事情貌似去年导师就跟我提过，当时说是让师弟们折腾一下，后来也不知道怎么的就没信儿了。我终于下定了决心，吃过晚饭，直奔5楼实验室。

装服务器的箱子很新，就那么静静的躺在实验室的角落，等着我的到来~动手搬，好重，搬不动！拆箱的时候还是蛮激动的，一个服务器机箱，几根数据线，还有两个金属支架（貌似往机柜上固定时用的）。说到机柜，不禁泪流满面，除了最上面一格，都满了！不会安装，而且，也举不动！

然后就是开动脑筋的时候了……最终，注意到了机房（机房就是一间小“黑”屋，不过真的好冷）唯一的一张小桌子，于是打扫干净，往桌子上一放（叫了同学一起搬的TT）。

回到工作室，把说明书都看了一遍，想着装什么系统呢？Windows直接排除，想着最好还是有个GUI，于是就选了Ubuntu，在网上下了Ubuntu桌面版，用U盘做系统盘。

再次直奔机房而去……

怀着激动的心情，插好了各种数据线（小桌子上的一个显示器也被我霸占了），摁下开机键。configuring memory……重启……configuring memory……重启……configuring memory……
这是什么鬼？！

百度、谷歌了好久，也没找到可行办法，暂时得出了一个内存条坏了的结论。

时间到了第二天早上，导师的办公室。跟导师描述了一下遇到的情况，我说这服务器是不是坏的？然后老师跟我说，可能硬件（内存）啥的缺了、坏了，让我打开机箱检查一下。

于是再次直奔机房而去……

于是很轻松的打开了机箱（这里还是要感谢老师的帮助，我本以为开箱需要拧开很多螺丝，结果服务器竟然是“滑盖”的，开锁，轻轻一抽，就打开了）

打开机箱的一瞬间，我就被这土豪的阵容惊呆了，两个CPU、12个内存插槽。这一刻我土鳖的气质暴露无遗。平缓了一下心情，把内存拔下来重新插进去，然后再开机，竟然成功了！

进入引导页，插入昨天做的盘系统盘，没反应，这又咋了？

想着可能这台服务器对U盘启动的支持不好，于是跑去买了光盘，要了DVD，店员竟然给了我700M的CD，我也没细看，就走了，然后又跑回来换。

回到工作室，借了同学电脑（这一刻觉得有光驱的电脑是那么美），很快就刻录好了。

于是再次直奔机房而去……

这一次，很顺利的就识别了光盘。当看到Ubuntu那标志性的紫色出现在屏幕上，这一刻，我泪流满面。

后面就是各种狂按下一步，插网线配置了网络，又配置了远程登录ssh。最终，我成功征服了这个大家伙！
